# E\*Trade

Redesigning a trading platform to modernize clunky UI and speed up trades.

## The Challenge

LLM notetaker prototype

## Quick Breakdown

### Team

Failenn Aselta — Full Stack Engineer, Product Designer

### Duration

1 week

### Tools

Figma, Photoshop, Illustrator

### Skills

Competitive Analysis, Data Visualization, Behavioral Design, Financial UI/UX, Wireframing

## Project Overview

### The Rundown

A simple UI with a low cognitive load is essential to a trading platform. E\*Trade's UI has constant complaints from users of site usability; many users are now switching to more modern platforms because small mistakes are costing them millions.

Mid-Fi Wireframes

Mid-fi Wireframes created with Figma

![](https://framerusercontent.com/images/YSPA18zJk946sR2NYuLbAaN8aI.png?width=1444&height=1028)

![](https://framerusercontent.com/images/VUDH0FuU7W2ms48ioo9nOoeiI.png?width=1440&height=1024)

Mid-Fi Wireframe 2 was selected for its right-side icon bar. While research shows users typically follow an F-shaped reading pattern, scanning the left side first, trading platforms reveal a different behavior: users often fixate on the lower left when analyzing data.

To align with this pattern, the icon bar was moved to the left, creating a more intuitive layout and allowing quicker, more efficient access to the watchlist and customization tools.

Considerations and Thoughts

![](https://framerusercontent.com/images/EA3vk2UkrC6sgezQx4vPnH5Ln3w.png?width=1919&height=1369)

Treating the E\*TRADE dashboard as a high-stakes data environment taught me that design anchored in clinical research produces measurably different outcomes than design anchored in intuition. The F-pattern finding was the clearest example: trading platforms contradict standard reading behavior entirely, and without the research, that layout decision never gets made. MIT AgeLab findings shaped choices that aesthetic judgment alone would have missed.

For the next iteration I would vibe code a working prototype to run a direct speed comparison between the original interface and the redesign. Behavioral outcomes need to be measured, not inferred.
